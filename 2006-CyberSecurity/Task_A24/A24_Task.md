How the internet talks:

A network protocol is a set of rules performed by computers to communicate. It's import for the purposes of security and reliable communication. 

There are various protocols such as TCP/IP, HTTP/HTTPS, FTP/SFTP.

For an example, i will cover TCP (Transmission Control Protocol). 

It is a foundational protocol used in other protocols like HTTPS. TCP is responsible for making sure data sent over the internet arrives correctly.
For example, a text message you send are recieved correctly. (Correct words, correct recipient)

TCP uses a 'Handshake' to begin data transfer. 
SYN: Your computer sends a 'lets talk' to the recipient.
SYN-ACK: The recipiant responds in acknowledgement
ACK: Your computer sends a message confirming this. 

This set-up allows the two devices over the network to be aware of the incomming traffic. Now it's prepared to send data the following happens:

Imagine a book being sent over the internet. 

The book sent is split into pages and mailed. On the mail a source and destination address is added, it's package number (correct page order) and a Checksum (A method for error detection)

Each page is mailed individually and can take different routes to the recipient. The recipient on receival, orders these packages and checks all of them arrived. 

In the event a page is missing, the recipient contacts the sender to send the page again so that the whole 'book' is recieved.

Why is this important?
TCP ensures reliable, and ordered delivery of data. This is crucial so that data is recieved completely and unaltered. If packets are lost, transformed or corrupted. It may indicate data has been exposed to hackers. In this small protocol we improve non-redupation and data integrity, ensuring users can trust the data was transmitted accurately and securely.
